COMMAND EXECUTION

CMD / PowerShell 双环境

同时支持 Windows CMD 和 PowerShell 两种命令环境，自动检测系统中可用的 PowerShell 版本， 优先使用 pwsh.exe (PowerShell 7+)，自动回退到 Windows PowerShell 5.1。 两种模式独立运行，各自维护工作目录状态。

  • /cmd exec - 安全执行 CMD 命令（自动转义特殊字符）
  • /cmd raw - 原样执行 CMD 命令（不转义）
  • /cmd pwsh - 执行 PowerShell 命令（推荐）
  • /cmd pwsh-raw - PS 原样执行
  • /cmd pwsh-admin - 以管理员权限执行 PS（触发 UAC）
  • 支持管道、重定向、通配符等所有 Shell 特性
  • 命令输出保持原始终端格式，自动适配系统编码
SYSTEM INFO

系统信息速查

不用切出游戏就能快速了解系统状态。CPU、GPU、内存、磁盘、网络、运行时间等信息一键查询， 游戏内实时监控系统运行状况。所有信息直接调用 Java 管理接口和系统 API 获取，数据准确可靠。

  • /cmd sysinfo - 系统信息总览（OS/CPU/内存等）
  • /cmd cpu - CPU 详细信息（型号、核心数、使用率）
  • /cmd gpu - 显卡信息查询
  • /cmd mem - 内存使用详情（堆内存/非堆内存）
  • /cmd disk - 磁盘空间使用情况
  • /cmd uptime - 系统运行时间
  • /cmd net - 网络接口信息
  • /cmd ping <host> - 测试网络连通性
  • /cmd env [key] - 查看环境变量
PROCESS

进程管理

游戏内直接查看和管理系统进程。可以列出所有正在运行的进程，支持关键词搜索筛选， 也可以直接终止指定 PID 的进程。终止进程操作需要二次确认，防止误杀。

  • /cmd ps - 列出所有运行中的进程
  • /cmd ps <关键词> - 按关键词筛选进程
  • /cmd kill <pid> - 强制终止指定进程（需确认）
  • 进程列表包含 PID、进程名、内存占用等信息
  • 危险操作需二次确认，30 秒超时自动取消
FILES

文件操作

游戏内直接浏览和操作文件系统。支持列目录、查看文件内容、写入文件、创建目录、 搜索文件等常用操作。文件大小超过 100KB 时自动限制显示行数，避免聊天栏刷屏。

  • /cmd ls [路径] - 列出目录内容（带图标和文件大小）
  • /cmd cat <路径> - 读取并显示文件内容（带行号）
  • /cmd write <路径> <内容> - 写入文本文件
  • /cmd mkdir <路径> - 创建目录
  • /cmd del <路径> - 删除文件/目录（需确认）
  • /cmd find <路径> <关键词> - 搜索文件（递归5层）
  • 支持相对路径和绝对路径，自动路径规范化
UTILITIES

实用工具集

内置多种实用系统工具，不用切出游戏就能完成常用操作。 截图、锁屏、睡眠、休眠、音量控制、定时关机等功能一应俱全。

  • /cmd screenshot - 一键截取屏幕并保存
  • /cmd lock - 锁定电脑屏幕
  • /cmd sleep - 系统进入睡眠（需确认）
  • /cmd hibernate - 系统休眠（需确认）
  • /cmd volume - 查看/设置/静音 系统音量
  • /cmd shutdown [秒] - 定时关机（需确认）
  • /cmd restart [秒] - 定时重启（需确认）
  • /cmd cancel-shutdown - 取消关机计划
  • /cmd open <路径/URL> - 打开文件或网页
  • /cmd clip <文本> - 复制文本到剪贴板
AUTOMATION

别名与脚本

频繁使用的命令可以设置别名，一键执行复杂操作。别名保存在配置文件中，重启游戏不丢失。 也可以直接运行 .bat 和 .ps1 脚本文件，让自动化任务在游戏内一键触发。

  • /cmd alias - 列出所有已定义别名
  • /cmd alias add <名称> <命令> - 添加新别名
  • /cmd alias remove <名称> - 删除别名
  • /cmd alias run <名称> - 执行别名命令
  • /cmd script run <路径> - 自动识别类型执行脚本
  • /cmd script bat <路径> - 执行 BAT 批处理脚本
  • /cmd script ps1 <路径> - 执行 PowerShell 脚本
  • 别名配置文件：config/mc2cmd_aliases.txt

安装步骤

01

安装 Fabric Loader

确保你已经安装了 Minecraft 1.20.1 和 Fabric Loader。 如果还没有安装，前往 fabricmc.net 下载 Fabric 安装器， 选择 1.20.1 版本进行安装。Fabric Loader 版本要求 ≥ 0.15.0。

Fabric Loader ≥ 0.15.0 · Minecraft 1.20.1
02

下载并放入 mods 文件夹

从上方任意渠道下载 SysCmd 模组文件（mc2cmd-1.3.jar）， 将其放入 Minecraft 游戏目录下的 mods 文件夹中。 通常路径为 %appdata%\.minecraft\mods\。如果没有 mods 文件夹，启动一次 Fabric 客户端后会自动创建。

%appdata%\.minecraft\mods\mc2cmd-1.3.jar
03

启动游戏并验证

启动 Minecraft 客户端，选择 Fabric 1.20.1 版本进入游戏。 创建或加入一个单人世界后，打开聊天框输入 /cmd sysinfo 来验证模组是否正常工作。 如果能看到系统信息输出，说明安装成功。

/cmd sysinfo → 显示系统信息 = 安装成功 ✓
04

开始使用

输入 /cmd help 查看完整命令列表。你可以使用 /cmd 执行 CMD 命令， 或 /cmd pwsh 执行 PowerShell 命令。执行前会有二次确认提示， 点击确认按钮后命令才会真正执行。

/cmd help → 查看所有命令 | /cmd exec dir → 列目录

第一层：二次确认机制

所有涉及系统变更的危险操作（如删除文件、终止进程、关机重启等）在执行前都会弹出确认提示， 显示即将执行的完整操作内容。你需要在 30 秒内输入 /cmd confirm 确认后， 操作才会真正执行。超时未确认的操作会自动取消，确保不会因为误触或遗忘而造成问题。

需要二次确认的操作：

  • ⚠ 删除文件/目录（/cmd del）- 防止误删重要数据
  • ⚠ 终止进程（/cmd kill）- 防止误杀关键进程
  • ⚠ 定时关机（/cmd shutdown）- 防止意外关机
  • ⚠ 定时重启（/cmd restart）- 防止意外重启
  • ⚠ 系统睡眠（/cmd sleep）- 防止意外中断游戏
  • ⚠ 系统休眠（/cmd hibernate）- 防止意外中断
  • ✓ 确认有效期：30 秒，超时自动取消
  • ✓ 随时可通过 /cmd cancel 取消操作

工作原理：当你执行危险命令时，模组不会立即执行，而是将操作保存到待确认队列中， 并在聊天栏显示确认提示。每个玩家的待确认操作是独立的，互不干扰。 只有输入 /cmd confirm 后，对应的操作才会被取出并执行。

第二层：危险命令拦截

模组内置了危险命令检测机制，能够自动识别可能对系统造成严重损害的高危操作并进行拦截。 即使你不小心输入了危险命令，模组也会在执行前将其拦截，从源头上杜绝灾难性的误操作。 这是比二次确认更底层的安全保障。

安全等级划分：

  • ✖ 禁止级：格式化磁盘、删除系统目录、破坏引导等 - 直接拦截
  • ⚠ 警告级：修改注册表、修改系统配置等 - 需二次确认 + 额外提醒
  • ⚠ 确认级：删除文件、终止进程、关机等 - 需二次确认
  • ✓ 安全级：查看信息、列目录、读文件等 - 直接执行

常见被拦截的操作：

  • ✖ 格式化命令（format、fdisk、diskpart 等）
  • ✖ 递归删除系统目录（C:\Windows、System32 等）
  • ✖ 破坏系统引导（bcdedit /delete、fixmbr 等）
  • ✖ 远程控制和后门类命令
  • ✓ 普通查看命令、信息查询、文件浏览等均可安全使用

第三层：服务端自动禁用

为了保障多人服务器的安全和公平性，SysCmd 在专用服务器（Dedicated Server）环境下会完全禁用所有功能。 这是一个硬编码的安全机制，模组在初始化时会检测运行环境，如果发现是服务端环境， 会直接跳过所有命令注册，确保任何人都无法在服务器上使用本模组。

不同环境下的行为：

  • ✓ 单人世界：全部功能正常可用（需 OP 权限 Level 4）
  • ✓ 对局域网开放：仅房主/主机可用，其他玩家无法使用
  • ✖ 专用服务器：功能完全禁用，命令不注册，无法使用
  • ✓ 权限要求：需要 OP 等级 4 才能使用所有命令

检测原理：模组在 Fabric 加载阶段会调用 FabricLoader.getInstance().getEnvironmentType() 检测当前环境类型。如果返回 SERVER，模组会立即设置禁用标记并返回，不进行任何命令注册。 同时在命令注册回调中也会再次检查是否为 DEDICATED 环境，双重保险。
